# 一、VMware Tools 安装

## 1. VMware Tools是什么？

VMware Tools 是 VMware 提供给虚拟机的增强工具。

作用：

提升宿主机 Windows 与 Linux 虚拟机之间的交互体验。

主要功能：

- 文件复制
- 文件拖拽
- 共享文件夹
- 自动调整分辨率
- 时间同步

---

## 2. 安装 VMware Tools 遇到的问题

在虚拟机中尝试安装 VMware Tools 时：

发现系统提示：

> 无法直接提供 VMware Tools 安装文件。


原因：

部分 VMware 版本不会直接提供传统 VMware Tools 镜像，需要用户自行下载 ISO 文件。

---

## 3. 解决方法

解决流程：

1. 根据提示进入网页下载 VMware Tools ISO 文件。

2. 将下载后的 ISO 文件加载到虚拟机。

3. 在 Linux 终端中进行安装。


安装过程主要包括：

- 挂载 ISO 文件
- 解压安装包
- 执行安装程序
- 完成配置


---

## 4. 学习总结

VMware Tools 不属于 Linux 核心知识。

需要掌握：

- 了解 VMware Tools 的作用
- 知道出现问题时如何解决


不需要：

- 背诵具体安装命令
- 记忆完整安装流程


重点理解：

> VMware Tools 是连接 Windows 宿主机和 Linux 虚拟机的重要工具。

# 二、VMware共享文件夹

## 1. 共享文件夹是什么？

共享文件夹用于实现：

Windows 主机

↓

Linux 虚拟机

之间的文件交换。


例如：

Windows中的文件：

```
test.cpp
```

可以通过共享文件夹在 Linux 中访问。

Linux中通常挂载位置：

```
/mnt/hgfs
```

---

## 2. 共享文件夹作用

主要用途：

- Windows保存资料
- Linux运行程序
- 两个系统之间传输文件


对于开发环境：

可以方便：

- 保存代码
- 编译程序
- 测试运行


---

## 3. 注意事项

如果 Linux：

运行在 VMware 虚拟机中：

可以使用：

- VMware Tools
- VMware共享文件夹


完成文件交换。


如果 Linux：

不是运行在虚拟机中，例如：

- 独立安装 Linux
- 云服务器
- 远程服务器


则需要使用远程方式：

例如：

- SSH
- SCP
- FTP


进行文件传输。

# 三、Linux目录结构

## 1. Linux目录结构特点

Linux采用：

> 层级式树状目录结构。


Linux所有文件和目录都从：

```
/
```

根目录开始。


目录结构：

```
/
├── bin
├── boot
├── dev
├── etc
├── home
├── usr
├── var
└── root
```

---

# 四、Linux“一切皆文件”思想

## 1. Linux中的文件思想

Linux的重要设计思想：

> Linux会将硬件设备也抽象为文件进行管理。


例如：

硬盘设备：

```
/dev/sda
```


设备文件：

```
/dev
```


Linux通过统一的文件接口管理：

- 普通文件
- 硬件设备
- 网络资源


---

## 2. 采用“一切皆文件”的优势

优点：

- 统一管理方式
- 简化系统操作
- 方便程序访问硬件


访问流程：

```
应用程序

↓

文件接口

↓

Linux内核

↓

硬件设备
```

# 五、Linux重要目录

## 1. / 根目录

作用：

Linux系统最顶层目录。


所有文件和目录：

都位于根目录下面。

---

## 2. /home 用户目录

作用：

保存普通用户文件。


类似 Windows：

```
C:\Users\
```


例如：

```
/home/user
```


常用于保存：

- 项目代码
- 文档
- 用户数据

---

## 3. /root

作用：

root管理员用户的主目录。


注意区别：

```
/
```

表示：

根目录。


```
/root
```

表示：

root用户目录。


---

## 4. /etc

作用：

系统配置文件目录。


configuration：

表示：

配置。


例如：

```
/etc/ssh/sshd_config
```


表示：

SSH服务配置文件。


主要用于：

- 网络配置
- 用户配置
- 软件配置

---

## 5. /usr

作用：

存放系统软件和程序。


例如：

```
/usr/bin
```


包含：

- 系统命令
- 软件程序


---

## 6. /bin

作用：

存放Linux基本命令。


例如：

- ls
- cp
- mv
- cat


---

## 7. /sbin

作用：

存放系统管理员使用的命令。


例如：

- reboot
- shutdown


---

## 8. /var

作用：

存放经常变化的数据。


例如：

日志文件：

```
/var/log
```


系统运行过程中产生的数据通常保存在这里。


---

## 9. /tmp

作用：

临时文件目录。


特点：

系统可能自动清理其中内容。


---

## 10. /sys

作用：

Linux内核提供的虚拟文件系统。


用于：

查看和管理硬件设备信息。


当前阶段：

了解即可。
